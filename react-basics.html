React Core

A clean, friendly introduction to modern React development using JavaScript and Vite.

1. Welcome to React!

React is a JavaScript library used to build user interfaces. Instead of writing massive HTML files, React allows you to split your website into small, reusable building blocks called Components.

What is a Component?

Think of a component as a custom HTML element. For example, a navigation bar, a button, or a profile card can all be separate components written in JavaScript using a special syntax called JSX (JavaScript XML), which allows you to write HTML directly inside your JavaScript code.

💡 Core Rule: Every React component function must start with a capital letter (e.g., App(), Button(), UserProfile()) and return some JSX.

2. Setting Up with Vite

Vite is a modern, ultra-fast tool that sets up a development environment for your React app. Gone are the days of waiting minutes for a server to load!

Step-by-Step Installation

  1. Install Node.js: Download and install the LTS version from nodejs.org. This gives you access to the command line tool npm.
  2. Open Your Terminal: Open VS Code and open a new terminal (Ctrl + ` or Cmd + `).
  3. Run the Setup Command: Run the following command to create your project: 
    npm create vite@latest my-first-react-app -- --template react
  4. Navigate inside your project folder: 
    cd my-first-react-app
  5. Install dependencies: Downloads the actual React files into your folder. 
    npm install
  6. Start the local server: 
    npm run dev
👉 After running npm run dev, look at your terminal. It will give you a local URL (usually http://localhost:5173). Open that in your browser to see your brand-new React application running live!

3. Understanding the Folder Structure 📁

When you open your new project in VS Code, you'll see a collection of files and folders. Let’s break down what they mean so you know exactly where to work.

my-first-react-app/
├── node_modules/       // Hidden giant folder holding downloaded project packages
├── public/             // Static assets like images or icons that don't change
├── src/                // THE MAIN KITCHEN: Where 99% of your coding happens!
│   ├── assets/         // Images, fonts, and logos used inside your components
│   ├── App.css         // Global styles specifically for the App component
│   ├── App.jsx         // Your primary Root component (the core structure of your site)
│   ├── index.css       // Global stylesheet for your entire website
│   └── main.jsx        // The entry point connecting React to the real HTML page
├── index.html          // The single real HTML page your app mounts into
├── package.json        // List of your project dependencies and scripts
└── vite.config.js      // Configurations for Vite

The Golden Rule of Vite & React:

  • src/main.jsx is the engine. It finds the <div id="root"></div> in your index.html and injects your React code right into it. You rarely need to touch this file.
  • src/App.jsx is your sandbox. This is where you will start adding text, buttons, routing, and creating your custom components!

4. Props in React

Props (short for "properties") are read-only data passed from a parent component down to a child component. They make your components dynamic and reusable.

Key Concepts

  • Props are read-only (immutable). A child component can never change the props it receives.
  • They look just like HTML attributes when passing them into a component.
  • We extract them inside the child component using JavaScript object destructuring.

Code Example: Greeting Component (App.jsx)

// 1. Define the Child Component
function Greeting({ name, age = 18 }) {
  return (
    <p>Hello, <strong>{name}</strong>! You are {age} years old.</p>
  );
}

// 2. Use it inside the Parent Component
function App() {
  return (
    <div>
      <Greeting name="Alice" age={25} />
      <Greeting name="Bob" /> // Falls back to default age: 18
    </div>
  );
}

Passing Elements as Props (Using children)

If you want to wrap a component around content, use the built-in children prop.

function Card({ title, children }) {
  return (
    <div style={{ border: '1px solid #ccc', padding: '1.5rem' }}>
      <h3>{title}</h3>
      {children}
    </div>
  );
}

// Usage
<Card title="User Profile">
  <p>This text goes right into the children prop!</p>
</Card>

Prop Directional Flow

Parent Component
→ passes data →
Child Component (read-only)
⚠️ Warning: Never try to modify a prop inside a child component. If you need data that changes over time, you need State!

5. The useState Hook

State represents data that can change over time based on user interactions (like typing in an input, opening a modal, or clicking a counter button). When state changes, React automatically re-renders the component to show the updated data on the screen.

Syntax Breakdown

We use the useState function provided by React. It returns an array with exactly two elements:

  • The current state variable: The value itself.
  • The setter function: A function used to update that value.
import { useState } from 'react';

function Counter() {
  // Declare state variable "count", starting at 0
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Current Count: {count}</p>
      // Always update state using the setter function!
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
}

State Interaction

State: 0
→ User clicks button →
State: 1 (UI auto-updates)

6. The useEffect Hook

The useEffect hook lets you synchronize your component with external systems. Think of it as a way to perform "side effects" like fetching data from an API, setting up a timer, or manually changing the browser document title.

The Dependency Array

The second argument passed to useEffect is an array [] that controls when the effect code runs:

  • [id]: Runs once when the component loads, and then re-runs only if the value of id changes.
  • [] (Empty array): Runs **only once** when the component first renders on screen.
  • No array at all: Runs on *every single render* (avoid this as it can cause performance issues).
import { useState, useEffect } from 'react';

function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  useEffect(() => {
    const fetchData = async () => {
      const res = await fetch(`https://jsonplaceholder.typicode.com/users/${userId}`);
      const data = await res.json();
      setUser(data); // Save fetched data to local state
    };

    fetchData();
  }, [userId]); // Re-run the fetch request if userId ever changes

  if (!user) return <p>Loading user profile...</p>;

  return <h1>Welcome back, {user.name}!</h1>;
}

7. Client-Side Routing (React Router v7) 🗺️

React builds Single Page Applications (SPAs). This means your browser never completely reloads a new HTML page from the server when clicking navigation links. Instead, React Router instantly swaps components on screen depending on the URL path.

How to Setup React Router

  1. Install the package in your project terminal: 
    npm install react-router-dom
  2. Wrap your app in the router provider inside your entry point file (src/main.jsx).

Step 1: Wire up main.jsx

import React from 'react';
import ReactDOM from 'react-dom/client';
import { BrowserRouter } from 'react-router-dom';
import App from './App.jsx';
import './index.css';

ReactDOM.createRoot(document.getElementById('root')).render(
  <React.StrictMode>
    <BrowserRouter>
      <App />
    </BrowserRouter>
  </React.StrictMode>
);

Step 2: Create Routes & Links inside App.jsx

Use <Routes> and <Route> to control what loads, and use <Link> instead of traditional <a href> tags to maintain zero-reload page navigation. 

import { Routes, Route, Link } from 'react-router-dom';

// Dummy page components for demonstration
const Home = () => <h2>Home Dashboard</h2>;
const About = () => <h2>About Our Application</h2>;

function App() {
  return (
    <div>
      <nav>
        <Link to="/">Home</Link> | 
        <Link to="/about">About</Link>
      </nav>

      <hr />

      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </div>
  );
}

export default App;
