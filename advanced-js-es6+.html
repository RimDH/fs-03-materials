1. Variables & Block Scope (Let vs. Const)

Forget old-school school tutorials that use var. Modern frameworks live or die by predictable variable scoping.

How to think about it: Use const for 95% of everything you write. It tells your teammates (and the browser) that this variable reference shouldn't change. Only use let when writing loops or counters that you explicitly plan to recalculate. 
// Use const by default. It stops you from accidentally erasing your data.
const layoutAlignment = "center"; 

// Only use let if you are manually updating a value later.
let clickCount = 0;
clickCount++;

2. Arrow Functions (The Framework Favorite)

You will see arrow functions everywhere in modern UI frameworks—especially when rendering lists or handling user clicks.

// Regular version with explicit return braces
const multiplyByTwo = (number) => { 
    return number * 2; 
};

// Ultra-short version (Implicit Return). No braces, no "return" keyword needed!
const tripleIt = x => x * 3;

// Sneaky trick: To implicitly return an object, wrap it in parentheses ()
const makeUserObject = username => ({ id: "123", name: username });

3. Destructuring + Rest / Spread

Data in frameworks is passed around inside large objects. Destructuring is like an extraction tool that pulls out exactly what you want in a single line of code.


const user = {
  name: "John",
  age: 25,
  country: "USA"
};

// Destructuring
const { name, ...restInfo } = user;

// Spread
const updatedUser = {
  ...user,
  job: "Developer"
};

// Rest (function parameters)
function printInfo(...args) {
  console.log(args);
}

printInfo("John", 25, "Developer");

console.log(name);        // John
console.log(restInfo);    // { age: 25, country: "USA" }
console.log(updatedUser); // { name: "John", age: 25, country: "USA", job: "Developer" }

4. Immutability (The #1 React Rule)

Never change an existing object or array directly!!!!

// WRONG WAY (Modifying the original array pointer)
const originalItems = [10, 20, 30];
originalItems.push(40); 

// RIGHT WAY (Creating a completely fresh clone with updates)
const freshItems = [...originalItems, 40]; // Uses the "spread" operator to copy

// Modifying a single item safely inside an array:
const updatedScores = originalItems.map(num => num === 20 ? 99 : num);

5. The Big Three Array Methods (Map, Filter, Reduce)

Say goodbye to classic for loops. In modern JS, we use these functional methods to transform data and turn it into UI components on the fly.

const itemsInCart = [
    { name: "Mechanical Keyboard", price: 120 },
    { name: "Gaming Mouse", price: 80 },
    { name: "Stickers", price: 5 }
];

// 1. .map() — Converts raw data arrays into visual HTML code blocks
const layoutHTML = itemsInCart.map(item => `<li>${item.name}</li>`);

// 2. .filter() — Keeps items that pass a specific test (Great for search bars)
const expensiveItems = itemsInCart.filter(item => item.price > 50);

// 3. .reduce() — Rolls up an entire array into a single calculation
const grandTotal = itemsInCart.reduce((currentTotal, item) => currentTotal + item.price, 0);

6. Optional Chaining & Nullish Coalescing

When you request data from a website or a server, it takes time to arrive over the internet. If your code tries to read that data before it has fully downloaded, JavaScript will freak out and crash your entire program. These safety operators act as a shield to stop that from happening.

// The ?. operator safely stops trying to read further if the data is currently missing
const userCity = serverResponse?.user?.address?.city ?? "No City Provided";

// Why use ?? instead of the old || operator?
const userScore = 0;
const resultsWithOR = userScore || 10;   // Returns 10 (Because it treats 0 as a broken or fake value)
const resultsWithNullish = userScore ?? 10; // Returns 0 (Because 0 is a completely valid score!)

7. Async / Await and Promises

When you use fetch() or any API call, JavaScript does not get the result immediately. Instead, it returns a Promise (an object that represents a value that will be available later)

A Promise has 3 states:
• Pending → still working
• Fulfilled → success (data received)
• Rejected → failed (error occurred)

async/await is just a clean way to work with Promises (do not block the rest of the code while waiting) 


async function loadUserData(userId) {
    try {
        // fetch() returns a Promise
        const response = await fetch(`/api/users/${userId}`);

        // check if Promise result is successful
        if (!response.ok) {
            throw new Error(`Request failed with status: ${response.status}`);
        }

        // another Promise (parsing JSON)
        const data = await response.json(); //{"name":"John","age":25}

        return data;
    } catch (error) {
        console.error("Error caught while handling Promise:", error);
        throw error;
    }
}

//another example
console.log("A");

async function test() {
  console.log("B");
  const response = await fetch("/api");
  console.log("C");
}

test();

console.log("D");

Key idea: Promises represent future results, and async/await lets you write them like normal step-by-step code.

8. ES Modules (Importing & Exporting files)

In modern development environments, you don't write your entire codebase inside a single massive document. Instead, you break everything down into tiny, bite-sized components and share tools between them.

// --- In a file named 'mathTools.js' ---
export const calculateTax = price => price * 0.15;
const globalSettings = { language: "en" };
export default globalSettings; // Every file can have exactly ONE default export

// --- In your main file 'App.js' ---
import systemConfig, { calculateTax } from "./mathTools.js";
